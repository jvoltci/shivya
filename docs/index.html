The Philosophical Shift

SHIVYA completely abandons classical consensus models in distributed state networks. Instead of forcing replicas to agree on a global sequential sequence of events, SHIVYA models history as a continuous causal manifold and transactions as geometric flows.

By applying the Hodge Decomposition Theorem on graph simplicial complexes, we isolate local transaction updates (gradient flow) from circular double spends and race conditions (curl flow). Replicas sync and merge states homeostatically without a central coordinator.

Quickstart & Local Simulation

1. To compile the workspace and run the test suite locally:

cargo check --workspace
cargo test --workspace

2. To run the background native edge daemon with the **Live WebOS Visualizer WebSocket Bridge** enabled:

cargo run --release -p shivya-cli -- start --port 8085 --visualize

3. To launch a secondary bootstrap node connecting to the P2P mesh:

cargo run --release -p shivya-cli -- start --port 8086 --peer 127.0.0.1:8085

4. Serve the repository locally (e.g. python3 -m http.server) in the root and visit http://localhost:8000/web/. The dashboard will automatically connect to your local WebSocket bridge on port 9002 and stream real-world physical metrics live!
